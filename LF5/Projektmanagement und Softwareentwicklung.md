# Projektmanagement und Softwareentwicklung

## Handlungssituation

> „Sie sollen in einem Projekt mitarbeiten, in dem eine Anwendungssoftware zur Verwaltung von Daten angepasst wird. Dazu erarbeiten Sie sich zunächst einmal grundlegendes Wissen über den Prozess der Softwareentwicklung im Allgemeinen."
> — *Westermann, IT-Berufe Grundstufe LF 1-5, S. 489*

---

## Was ist ein Projekt?

Ein Projekt ist ein Vorgehen, das durch die **Einmaligkeit** seiner Bedingungen gekennzeichnet ist:

- **Zielvorgaben** - klares Ergebnis definiert
- **Zeitliche Begrenzung** - fester Start- und Endtermin
- **Finanzielle Begrenzung** - Budget festgelegt
- **Personelle Begrenzung** - Team zusammengestellt
- **Projektspezifische Organisation** - eigene Struktur

### Projektphasen

```
Initiierung → Planung → Durchführung → Kontrolle → Abschluss
```

### Das magische Dreieck

```
        Qualität
       /        \
      /          \
   Zeit ———————— Kosten
```

Alle drei Faktoren beeinflussen sich gegenseitig. Wird einer verändert, wirkt sich das auf die anderen aus.

## SMART-Regel für Projektziele

| Buchstabe | Bedeutung     | Erklärung                        |
| --------- | ------------- | -------------------------------- |
| **S**     | Spezifisch    | Eindeutig und klar formuliert    |
| **M**     | Messbar       | Ergebnis ist überprüfbar         |
| **A**     | Attraktiv     | Motivierend und erreichbar       |
| **R**     | Realistisch   | Mit vorhandenen Mitteln machbar  |
| **T**     | Terminiert    | Zeitrahmen festgelegt            |

## Softwareentwicklung

Die Konzeption und standardisierte Umsetzung von Softwareprojekten. Wichtige Aspekte:

- Wartbarkeit
- Softwarearchitektur
- Dokumentation
- Qualitätssicherung

### Phasen des Softwareentwicklungsprozesses

| Phase           | Beschreibung                                        |
| --------------- | --------------------------------------------------- |
| **Analyse**     | Anforderungen erfassen (Lastenheft)                 |
| **Entwurf**     | Architektur und Design festlegen (Pflichtenheft)    |
| **Implementierung** | Programmierung und Codierung                   |
| **Test**        | Fehler finden und beheben                           |
| **Deployment**  | Auslieferung und Installation                       |
| **Wartung**     | Pflege, Updates, Bugfixes                           |

### Projektinstanzen

- **Auftraggeber** - definiert Anforderungen
- **Projektleiter** - koordiniert das Team
- **Projektteam** - setzt das Projekt um
- **Stakeholder** - alle Betroffenen (Kunden, Nutzer, Management, ...)

## Wissenstreppe nach North

```
Zeichen → Daten → Informationen → Wissen → Können → Handeln → Kompetenz
```

| Stufe           | Beschreibung                                     |
| --------------- | ------------------------------------------------ |
| **Zeichen**     | Einzelne Symbole ohne Bedeutung (A, 3, #)        |
| **Daten**       | Zeichen mit Syntax (23.05.2026)                  |
| **Informationen** | Daten mit Bedeutung ("Heute ist der 23. Mai") |
| **Wissen**      | Vernetzte Informationen mit Erfahrung            |
| **Können**      | Anwendung von Wissen                             |
